# 混天DB 安全设计

## 威胁模型

混天DB 面向企业安全审计场景，默认假设以下威胁：
- 网络窃听（被动攻击者）
- 中间人攻击（MITM）
- 未授权数据访问（内部威胁）
- 数据篡改（完整性攻击）
- 重放攻击

## 安全层次

### 1. 传输层安全 — TLS 1.3 + ExtDH

```
协议版本:    TLS 1.3（最低要求）
密钥交换:    ECDHE with P-521 curve
对称加密:    AES-256-GCM
哈希:        SHA-384
证书验证:    mTLS（双向认证）
```

**P-521 ECDHE 密钥交换**：
- 比 P-256 更强的安全边界（~256-bit 等效安全性）
- NIST 推荐曲线中安全性最高的标准曲线
- 每次连接独立生成临时密钥对（前向保密）

### 2. 静态数据加密

所有持久化数据均经 AES-256-GCM 加密：

```
主密钥 (32B)
    ↓ HKDF-SHA256
    ├── WAL 加密密钥 (32B)
    ├── 元数据加密密钥 (32B)
    └── Parquet 加密密钥 (32B)
```

- 密钥从不写入磁盘（仅存于环境变量/密钥管理服务）
- 支持密钥轮换（version 字段在加密头中）
- 每个 WAL 条目使用独立随机 Nonce

### 3. 认证方式

| 方式 | 适用场景 | 说明 |
|------|----------|------|
| SCRAM-SHA-256 | PostgreSQL 客户端 | RFC 7677，密码不传输明文 |
| mTLS | 服务间通信 | 双向证书验证 |
| JWT | REST API / Portal | 无状态 Token，24h 过期 |

### 4. 基于角色的访问控制 (RBAC)

```
Admin (4级)
 ├── 所有权限
 │
Auditor (3级)
 ├── 读取事件
 ├── 创建快照
 └── 取证追踪
 │
Writer (2级)
 ├── 写入事件
 └── 读取事件
 │
Reader (1级)
 └── 仅读取
```

权限检查在查询执行前进行，越权操作返回 `403 Forbidden`。

### 5. 审计日志

所有敏感操作自动记录：
- 认证尝试（成功/失败）
- 权限变更
- 快照创建/删除
- 配置修改

审计日志本身也存储在混天DB中，不可删除。

## 安全最佳实践

### 生产部署检查清单

- [ ] 替换默认 JWT 密钥（至少 32 字符）
- [ ] 使用 CA 签名的 TLS 证书（非自签名）
- [ ] 配置 mTLS 客户端证书验证
- [ ] 设置 DB_ENCRYPTION_KEY 并定期轮换
- [ ] 启用防火墙（仅开放 5408/5000）
- [ ] 配置网络隔离（VPC/安全组）
- [ ] 定期备份数据目录
- [ ] 监控异常访问模式

### 密钥管理

```bash
# 生成 AES-256 密钥
openssl rand -base64 32

# 生成 JWT 密钥
openssl rand -base64 64

# 生成 P-521 ECDSA 私钥
openssl ecparam -genkey -name secp521r1 -out server.key
```

### 网络安全

- 生产环境使用反向代理（nginx/Caddy）终止 TLS
- 启用速率限制（1000 requests/sec per IP）
- 配置 CORS 白名单
- 使用网络策略限制 Pod 间通信（Kubernetes）
